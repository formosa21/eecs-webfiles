    • Paradigm: Parent Paradigm: What's it about:
    Imperitive Explicitly describe actions of a program. Uses sequence of statements to modify program's "state". (e.g. C++, java)
    Declarative Programs decribe what they want not how to get it. Tends to lack side effects. (SQL, Xquery, Haskell)
    Structure Imperitive, Declarative Get rids of the gotos. Use nice block of codes.
    Procedural Imperitive, Declarative, Structure Based on Procedures(functions) which can typically be called from anywhere in the code.
    Functional Imperitive, Declarative, Structure Same as Procedural but with no state.
    Object Oriented Imperitive Data structures form objects which contain methods(functions) and attributes. They are created/instantiated from blueprints/classes.
    Logic Declarative Engine answer questions based on facts and rules.
